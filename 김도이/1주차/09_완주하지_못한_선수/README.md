## 3. 풀이 설명
### 💡 아이디어 도출 과정
> 처음에 그냥 정말 쉬운 문제라고 생각했다. participant를 반복문 돌려서, completion의 요소들을 포함하면 participant에서 해당 String을 제거하면 되는거 아닌가?
> 라고 생각했는데, 우선 내가 생각한 로직이 java에서 그대로 동작하는 메서드들도 없고, 시간복잡도가 높아서 시간초과위험이 있다고 chatGPT가 알려줬다.<br>
> 결국 내가 원하는 대로 잘 안돼서 이번에도 chatGPT를 사용했다..<br>

### ⚙️ 구현 로직의 동작 과정
> 1. 참가자 이름을 key로 HashMap에 저장하고, 값을 getOrDefault(key, 0)+1 처리한다.
> 2. 완주자 이름을 key로 HashMap에서 찾으면, 값을 -1 처리한다.
> 3. HashMap에서 값이 0이 아닌 key를 바로 반환한다(답은 무조건 한 명이라고 문제에 나와있기 때문에)

### ✅ 엣지 케이스에 대한 고려 사항
> 반환값이 String인 함수로 만들어져 있어서 무조건 return <String>으로 했어야 했는데, 기본으로 있는 return문을 없애거나, 0을 주거나,<br>
> 하니깐 에러나버려서, return ""; 처리를 하긴 했는데, 이렇게 우선 돌아가게만 처리해두는 게 맞나? 라는 생각을 했다.


## 4. 어려웠던 점 및 해결 과정
> 동명이인이 존재할 수 있다고 해서 중복제거하는 자료구조들은 전혀 생각지도 못하고 있었다.<br>
> 이번 기회로 HashMap.getOrDefault(key, 0)+1 하는 방법을 꼭 외워야겠다고 생각했다. 먼저 맵에 해당 key가 있나 확인해보고,
> 없으면 Default기본값으로 0처리 후 -> +1 / 있으면 값을 찾아오고 해당 값에서 +1 처리로 제대로 돌아간다.
